# 局域网图片上传与共享系统部署说明

## 系统概述

本系统实现了一套「局域网内图片上传+URL自动归档+共享访问」功能，核心目标是：前端上传图片后，后端自动生成局域网可访问的图片URL，将URL追加写入文本文件，并提供固定网址供局域网内所有设备查看/访问所有URL。

## 技术栈

- 后端：Spring Boot
- 前端：Vue + Element UI
- 存储：本地文件系统

## 部署步骤

### 1. 环境准备

#### 后端环境
- JDK 1.8 或以上
- Maven 3.6 或以上

#### 前端环境
- Node.js 12 或以上
- npm 6 或以上

### 2. 后端部署

#### 2.1 配置修改

修改 `src/main/resources/application.yml` 文件：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/graduation_framework?serverTimezone=GMT%2B8&useUnicode=true&characterEncoding=utf8&useSSL=false
    username: root
    password: 123456
    driver-class-name: com.mysql.cj.jdbc.Driver
  servlet:
    multipart:
      max-file-size: 100MB
      max-request-size: 500MB
  jackson:
    date-format: yyyy-MM-dd HH:mm:ss
    time-zone: GMT+8

server:
  port: 8888  # 可以根据需要修改端口号

mybatis-plus:
  mapper-locations: classpath:mapper/*.xml
  configuration:
    log-impl: org.apache.ibatis.logging.stdout.StdOutImpl

files:
  upload:
    path: D:\competition\graduation-framework\file\  # 图片存储路径，根据实际情况修改
```

#### 2.2 编译打包

在项目根目录执行：

```bash
mvn clean package -DskipTests
```

#### 2.3 运行

执行打包后的jar文件：

```bash
java -jar target/graduation-framework-1.0.0.jar
```

### 3. 前端部署

#### 3.1 安装依赖

在 `vue` 目录执行：

```bash
npm install
```

#### 3.2 编译打包

在 `vue` 目录执行：

```bash
npm run build
```

#### 3.3 部署

将 `dist` 目录下的文件部署到Web服务器（如Nginx、Apache等）。

### 4. 防火墙配置

确保服务器防火墙开放了配置的端口（默认8888），允许局域网内其他设备访问。

## 使用说明

### 1. 访问系统

在浏览器中访问：`http://服务器IP:端口`

### 2. 上传图片

1. 点击「图片处理」菜单项
2. 点击「选择图片上传」按钮
3. 选择要上传的图片（支持jpg、png、webp格式，最大10MB）
4. 等待上传完成，查看生成的局域网URL

### 3. 访问共享URL

1. 在「共享访问」区域复制共享地址
2. 在局域网内其他设备的浏览器中粘贴该地址
3. 查看所有上传的图片URL列表
4. 点击URL可直接访问图片

## 测试用例

### 1. 上传测试
- 测试上传不同格式的图片（jpg、png、webp）
- 测试上传接近10MB的大图片
- 测试上传重复文件名的图片（应生成不同的UUID文件名）

### 2. URL访问测试
- 测试复制生成的URL到浏览器中访问
- 测试在不同设备上访问URL

### 3. 局域网共享测试
- 测试在局域网内另一台设备上访问共享地址
- 测试点击共享页面中的URL是否能打开图片

## 额外功能

1. **上传进度条**：显示图片上传进度
2. **URL复制**：一键复制生成的URL到剪贴板
3. **图片预览**：上传后预览图片效果
4. **共享页面**：一键打开共享页面

## 注意事项

1. 确保服务器和其他设备在同一局域网内
2. 确保服务器防火墙开放了相应端口
3. 定期清理 `file` 目录，避免存储空间不足
4. 本系统仅在局域网内使用，不建议暴露到公网

## 故障排查

1. **上传失败**：检查文件大小是否超过限制，格式是否支持
2. **URL无法访问**：检查服务器IP是否正确，防火墙是否开放端口
3. **共享页面无法访问**：检查服务器是否正常运行，网络连接是否正常

## 性能优化

1. 图片上传时生成唯一UUID文件名，避免重复
2. 文件写入使用BufferedWriter提升效率，避免频繁IO
3. 静态资源访问添加7天缓存头，优化加载速度
4. 后端上传完成后及时释放文件句柄，避免内存泄漏
5. 前端上传完成后清空File对象引用，释放内存
